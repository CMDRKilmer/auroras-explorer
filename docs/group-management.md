# Group Management

## Overview

Group management allows users to create, edit, and select FIO groups. Users can link their FIO groups to the platform, enabling group-level tools such as contract tracking, member management, price watching, and production planning.

## Architecture

### File Structure

```
src/
├── routes/group/
│   ├── index.tsx                          # /group/ - Group list page route
│   └── {-$groupId}/
│       ├── route.tsx                      # /group/:groupId - Layout with group selector fallback
│       ├── contracts/index.tsx
│       ├── members/index.tsx
│       ├── plan/index.tsx
│       └── price-watch/index.tsx
├── components/
│   ├── pages/group/
│   │   ├── group-list-page.tsx            # Main group list page
│   │   └── management/
│   │       ├── group-card.tsx             # Single group card with tool links
│   │       ├── group-empty-state.tsx      # Empty state when user has no groups
│   │       ├── group-form-dialog.tsx      # Create/edit group dialog (TanStack Form)
│   │       └── group-selector.tsx         # Inline group selector for routes without groupId
│   ├── common/breadcrumb/
│   │   └── app-breadcrumb.tsx             # App-wide breadcrumb navigation
│   └── auth/require-auth.tsx              # Auth guards (RequireAuth, RequireGroupAuth)
├── hooks/
│   └── use-navigates.tsx                  # Navigation items (useNavigates, useGroupTools)
└── lib/query/
    └── group.ts                           # Query hooks (myGroupsQuery, etc.)
```

### Data Flow

```
FIO API ──> POST /api/group ──> groups table (internal ID: typeid)
                                user_groups table (group_id = fioGroupId)

GET /api/my/groups ──> user_groups (by username) ──> groups (by id) ──> client
```

- `groups.id`: Internal ID generated by `typeid` (e.g., `group_xxxxxxxxxxxx`)
- `groups.fio_group_id`: FIO group ID (e.g., `873386`), used in URLs
- `user_groups.group_id`: References the FIO group ID, not the internal ID
- URL routes use `fioGroupId` as the `groupId` param

## Group List Page

**Route:** `/group/`

Displays all groups the current user belongs to. Each group card shows:
- Group name and FIO group ID
- Quick links to all group tools (Members, Contracts, Price Watch, Plan)
- Edit button (visible on hover)

A "New Group" button opens the create dialog.

**Auth:** Wrapped in `RequireAuth` — requires login but no specific group membership.

## Group Form Dialog

**Component:** `GroupFormDialog`

Uses `@tanstack/react-form` with `onSubmit` validators. Fields:

| Field | Type | Notes |
|-------|------|-------|
| Group Name | text | Required |
| FIO Group ID | text | Required; disabled on edit |
| Use my FIO API token | checkbox | Toggles token input visibility |
| FIO API Token | text | Required when "use my token" is unchecked |

Calls `POST /api/group` which validates:
1. The FIO API token is valid (via FIO API)
2. The current user is an admin of the FIO group
3. Upserts the group (on conflict by `fio_group_id`)

The dialog is keyed by `editingGroup?.id` in the parent, so it resets state when switching between groups.

## Group Selector

**Component:** `GroupSelector`

Shown inline at `/group/:groupId/*` routes when `groupId` is missing or is the placeholder value `_`.

**How it works:**

1. Sidebar nav links use a placeholder `groupId` of `_` when no group is currently selected (i.e., user is not on a group route)
2. The route layout at `route.tsx` checks if `groupId === '_'` and renders `GroupSelector` instead of the child route
3. `GroupSelector` detects which group tool the user intended to visit (via `useMatchRoute`) and lists available groups as buttons
4. Clicking a group navigates to the intended tool with the correct `groupId`
5. A "Go to Group List" link is provided for managing groups

## Breadcrumbs

**Component:** `AppBreadcrumb` (rendered in `SiteHeader`)

Breadcrumb structure varies by route type:

**Group sub-routes** (e.g., `/group/873386/contracts/`):
```
Groups > GroupName > Group Contracts
```
- "Groups" links to `/group/`
- Group name links to `/group/:groupId/members/`
- Current tool name is the active (non-clickable) page

**Non-group routes** (e.g., `/shipment/`):
```
Shipment
```

Route labels are derived from `useNavigates()` and `useGroupTools()` — no separate label mapping is maintained. Group names are resolved from the `myGroupsQuery` cache by matching `fioGroupId`.

## Navigation Integration

**File:** `src/hooks/use-navigates.tsx`

Exports two hooks:
- `useGroupTools()`: Returns group tool items with parameterized URLs (`/group/{-$groupId}/members/`, etc.)
- `useNavigates()`: Returns all navigation items (explorer tools + group tools), consumed by the sidebar and breadcrumbs

The sidebar (`NavMain`) passes `groupId` from the current route params to all links. When no `groupId` is available, it uses `_` as a placeholder, which triggers the group selector on navigation.
